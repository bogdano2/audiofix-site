Fix Audio on Linux

We're building a guide for fixing audio device switching on Linux covering both major audio systems. The same four-phase approach applies: identify, set default, disable unused, check apps.

What to expect

Linux has two mainstream audio systems, and the guide will cover both. Which one you need depends on your distribution and how recently it was installed.

  • PulseAudio (pactl) — the established audio server used by Ubuntu (pre-22.10), Debian, and many others. You'll learn to list sinks, set defaults, and manage per-application streams from the terminal
  • PipeWire (wpctl) — the modern replacement shipping by default on Fedora, Ubuntu 22.10+, and Arch. Same concepts, different commands — we'll cover the WirePlumber interface for device and stream control
  • The four-phase framework — identify your devices, set the correct default, disable unused endpoints, and verify per-app routing. Same structure as the Windows guide, adapted for Linux tooling
In the meantime: The concepts in our guide to why audio breaks apply to every operating system. Understanding why your OS switches devices is the first step to fixing it, regardless of platform.
