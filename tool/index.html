The AudioFix Script — Automate the Fix

This PowerShell script does everything you learned in the guide — automatically and interactively. It follows the same three steps: find your devices, test them, and lock it down.

Haven't read the guide yet? Start with why it happens to understand what the script is doing and why.

Three steps, fully automated

Each step mirrors a page from the guide — the script just does it faster.

Step 1 — Find your devices

Lists every audio device Windows is managing — the same view you'd see in Sound Settings. Flags impostors like Stereo Mix and phantom HDMI outputs so you know what you're looking at.

Step 2 — Test your devices

Plays a test tone through each output device one at a time — you listen for which one is your real speakers. For input, shows a live mic level meter while you speak. Same test as in the guide, just automated.

Step 3 — Lock it down

Sets your device as default across all three audio roles (Console, Multimedia, Communications) in one shot, then disables unused devices via PnP — the single most effective way to prevent future drift.

How it works under the hood

Every component is deliberately chosen from what Windows already ships. The design philosophy is zero external dependencies — no third-party modules, no NuGet packages, no downloaded DLLs. Every dependency you don't add is an attack surface that doesn't exist and a privacy policy you don't have to read.

Windows APIs and built-in tools the AudioFix script uses, and what each one does.
Technology What it does
Core Audio API
IMMDeviceEnumerator, IAudioMeterInformation		 The same interfaces Sound Settings uses internally. Enumerates every audio endpoint and reads live mic levels during the Find and Test steps.
IPolicyConfig COM interface Sets the system default across all three Windows audio roles (console, multimedia, communications) in a single call — something the Settings UI doesn't even expose.
In-memory WAV generation
System.Media.SoundPlayer		 Generates test tones entirely in memory. No temp files are written to disk, no audio files are downloaded.
PnP cmdlets
Get-PnpDevice, Disable-PnpDevice		 Built into Windows. Disables unused audio devices at the hardware level so Windows can't silently fall back to them.
Embedded C# via Add-Type All COM interop is compiled inline from C# source code visible in the script. Single auditable text file — no external binaries.
Zero network requests. No third-party code is downloaded, loaded, or executed. The script makes zero network requests. This eliminates supply-chain risk entirely — what you download is what runs, with no moving parts beyond the Windows APIs already on your machine.
